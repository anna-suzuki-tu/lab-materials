LAB TOOLS — QUARTER PLAN

Quarter Plan

90日で研究を設計する — 進捗報告ではない。思考構造の可視化である。
Quarter Plan は、3ヶ月ごとに「問い・目標・逆算設計」を言語化するドキュメントである。 自律と構造的思考のための習慣。ミーティング前だけ動くのは、自律が弱いサイン。

なぜ Quarter Plan なのか

研究はマラソンです。しかし1年単位の計画は長すぎて構造のズレに気づけず、週単位の予定は短すぎて戦略になりません。90日は、構造のズレが早期に見え、かつ戦略が立てられる単位です（Lab OS 05「学年別のゴールと計画」）。

もう一つの理由は、学生の1年が研究だけでは構成されていないことです。院試・中間審査・修論提出・卒論発表——動かせない外部イベントが研究時間を圧迫します。だから計画は積み上げではなく、固定イベントからの逆算で設計します。

そして最も大事なこと：Quarter Plan は教員に見せるための進捗報告書ではありません。自分の思考構造を自分で可視化する文書です。教員を待たずに「自分の現在地」を言語化する——この練習の積み重ねが、自律した研究者をつくります。

Quarter Plan が果たす3つの役割

  1. 逆算の設計図 — 固定イベントから逆算して90日を設計する。Quarter Goal を決め、月ごとのマイルストーンに割り、達成の目安まで書く。研究が計画どおりに進まないのは正常——ズレが見えることに価値がある
  2. 個別面談の燃料 — Quarter Plan は面談の起点です。Plan があってこそ対話が深まる。空で来ない。面談が問うのはただ一つ、「あなたはどこへ向かっているか？」
  3. 振り返りの装置 — Quarter 末に「達成できたこと・できなかったこととその理由・新たに生まれた違和感」を正直に書く。この振り返り（Section 0）が次の Quarter への橋渡しになる。違和感は探究の起点です（Lab OS 01）

テンプレートの構成 — 6つのセクション

タイトルは「名前 学年_Quarter」（例：Anna Suzuki M1_Q1）。書くのは以下の6つです。

Section内容
0｜前Quarter振り返り前回の Goal と Milestones の評価・達成できたこと・できなかったこととその理由・新たに生まれた違和感（最初の Quarter は不要）
1｜位置づけ卒業・修了までのロードマップ上で、この Quarter はどこか。現在の Core Question（1文）。今どの Phase か（Idea / Framing / Figures / Draft / Submission）
2｜Quarter Goalこの3ヶ月で何を達成するか——英語1文＋その理由・背景＋成功の定義
3｜Monthly Milestones月ごとの逆算テーブル：やること・達成の目安・（Quarter末に）評価
4｜習慣ログResearch Log（毎日）・Thinking Log（毎週月曜）・Paper Card 更新（毎週）の現状
5｜研究室へのフィードバックミーティング体制・研究室運営・環境への意見や提案

Phase と習慣ログの項目が Paper Card や Research Log と同じ言葉で書かれていることに注目してください。Quarter Plan は独立した書類ではなく、Lab OS の道具たちを90日の時間軸の上に並べる役割を持っています。

使い方のリズム — 個別面談との連動

  1. 面談の前日までに、自分で記入して提出する — 教員の入力を待たずに現在地を言語化するのが目的
  2. 面談では Section 2・3 を中心に議論する — Quarter Goal の妥当性と逆算マイルストーンを一緒に確認・修正する。報告の場ではなく、判断の場
  3. Quarter 末に Section 0（振り返り）を書く — 達成・未達・持ち越し・違和感を正直に。次の Quarter Plan の起点になる

個別面談は四半期に一度。やることは研究軸の確認・成長の振り返り・キャリア設計・状態確認（迷い・負荷）。やらないことは技術的議論・原稿の細部チェック・日常的な進捗報告——それらは Team Meeting の仕事です（Lab OS 04：場の目的を混ぜない）。

1年のリズム — 4つの Quarter

Quarter時期基本テーマ
🌸 Q14–6月年間設計・逆算ロードマップ
🏖️ Q27–9月仮説検証（B4は院試と並走）
🍂 Q310–12月図と Draft・学会/審査
❄️ Q41–3月提出・発表・次年度設計

1年を4つの90日に分け、固定イベントから逆算して設計する——このリズム自体が、Lab OS の「マラソンをスプリント設計で走る」の実装です。

ロードマップ — Quarter Goal は白紙から立てない

Quarter Plan の Section 1 で最初に参照するのが、学年ごとのロードマップです。ロードマップは進捗管理表ではありません。「いつ、何のために、何をするか」の構造を理解するための地図——自分の Quarter がこの地図のどこにあるかを確かめてから、Quarter Goal を立てます。

学年ミッション面談で確認されること
B41サイクルを経験する研究を1文で説明できるか。詰まりを言語化できるか
M1研究を設計するFraming は進んでいるか。2月セミナーから逆算できているか
M2研究を完走する1月提出への逆算は現実的か。図は凍結できるか。何から逃げているか
D1軸を定義する研究軸を1〜2文で書けるか。追わないトピックを宣言できているか
D2軸を証明する論文群は一貫しているか。ノイズ論文を切れているか。投稿計画はあるか
D3言語化して出す博論背骨3文は書けるか。削る対象を宣言できるか

🗓 B4ロードマップ — 全体像を1回経験する年

B4のゴールは「完璧な研究」ではありません。研究の全体像を1回経験することです。固定イベント（5月上旬 TOEIC申込 → 7月上旬 院試出願 → 8月 院試 → 2月 卒論発表 → 3月 卒論提出）から逆算します。

  1. Q1（4–6月）違和感に触れ、問いを探る — テーマに触れ、違和感を溜める。評価せずに「何がおかしいか」を書き出す。Core Question の初稿を書いてみる（完璧でなくてよい）。TOEIC 申込を忘れない
  2. Q2（7–9月）院試＋仮説を立てる — 院試期間中も研究を「死なせない」：週に最低1回は研究に触れる日を作り、院試後最初の1週間に何をするかを今決めておく。9月から本格的なデータ収集を開始
  3. Q3（10–12月）図を作り、主張を試す — 空図を先に設計し、仮結果で「主張が崩れる場所」を探す。証明しにいかない。12月末までに主要データの第一弾
  4. Q4（1–3月）卒論発表・提出 → M1へ — IMRAD で書く。初稿は早めに。「自分はこの問いを追った」と言える状態で発表に臨み、M1に持ち越す問いと仮説を整理する

よくある崩れ：「院試があるから」と研究を完全に止める／分からないことを黙る／完璧な図を待ってから手を動かす。Quarter Plan はこれを防ぐための装置です。

🗓 修士2年間ロードマップ — M1で設計し、M2で完走する

M1は「設計する」年。ミッションは、Framing を完成させ、M1 2月の分野横断セミナーで専門外の人に研究の価値を伝えられる状態にすることです。

  • Q1 — 違和感を問いに変える。「Compared to X, we Y」の比較対象を3本程度に絞り、Core Question 初稿を書いて毎週書き直す
  • Q2 — Framing を固定する。Core Question が動かない状態にし、Story Map と空図を設計する
  • Q3 — 空図に対応する実験・解析を実施し、仮結果で主張が成立するか確認する
  • Q4 — 2月セミナーで価値を伝え、直後にM2の年間設計（修論提出から逆算）を行う

M2は「完走する」年。言い訳を作らない。修論提出（1月末）から逆算した死守ラインが決まっています：

7月末 図の凍結 → 8月末 Draft-A → 10月末 Draft-B → 11月末 中間審査 → 12月末 チーム内提出 → 1月 修論提出 → 2月 発表

よくある崩れ：まだ図を増やそうとする（→「既存の図で代替できないか」を先に問う）／書くより実験を優先する（→ 図が凍結されたら翌日から Draft）／タイムラインを直視しない（→ 毎月1回、逆算表を更新して現実とのギャップを見る）。

🗓 博士3年間ロードマップ — 定義する・証明する・言語化して出す

博士の3年間は、学年ごとにミッションが明確に異なります。

  1. D1：軸を「定義する」年 — 「私は X という問いを Y という方法で探究する」を1〜2文で書く。広げるのではない。追わないトピックリストを作り、広げたい衝動を封印する。軸を定義せずに D2 へ進むのは、砂の上に論文を積むこと
  2. D2：軸を「証明する」年 — 論文群に一貫した軸があることを、論文という形で外に出して証明する。最大のリスクはノイズ論文を切れないこと。判定基準は「この論文を博論から抜いても、軸の主張は成立するか？ → Yes ならノイズ」。最重要論文を今期中に投稿する条件を、今書く
  3. D3：軸を「言語化して社会に出す」年 — ミッションは削り、言語化し、提出する。追加しない。博論の背骨（3文：問題→アプローチ→貢献）を毎月書き直し、11月の予備審査 → 1月初旬の提出・発表へ。「提出できるが完璧ではない」の定義を、完璧を待たずに今書く

よくある崩れ：軸が定まらないまま実験だけ進む（D1最大の罠）／投稿・削る決断の先延ばし（→「残したい気持ちがあるものほど削る候補」）。

学年が変わっても、変わらないもの

どのロードマップにも、同じ習慣の表が載っています。これはタスク管理ではなく、探究の姿勢を保つための習慣です（Lab OS 06）。

習慣頻度目的
Research Log毎日思考の痕跡を残す
Thinking Log毎週月曜週単位で問いを更新する
Paper Card 更新毎週論文の現在地を把握する
Core Question 書き直し毎月問いが変化していることを記録する

Quarter Plan の Section 4（習慣ログ）は、この習慣が回っているかを四半期ごとに自分で点検する場所です。

使い始める

研究室メンバー：Quarter Plan は Notion のデータベースで管理しています。新しい Quarter が始まったら、DB 内のテンプレートから作成し、タイトルを「名前 学年_Quarter」に変えて記入してください。個別面談の前日までに提出がルールです。

🗓 Quarter Plan DB を開く（メンバー限定）

※ DB の閲覧・編集は研究室メンバーのみ。ログインが必要です。学年別ロードマップの全文も研究室 Notion にあります。

研究室外の方へ：構成はこのページに書いたとおりです。「固定イベントからの逆算」「90日単位の Goal と月次マイルストーン」「Quarter 末の正直な振り返り」——この3点が本質で、ツールは何でも再現できます。

このページのまとめ：Quarter Plan は進捗報告ではなく思考構造の可視化 ／ 固定イベントから逆算して90日を設計する ／ 面談の前日までに自分で書いて提出（Plan は面談の燃料）／ Quarter 末の振り返りが次の起点 ／ Goal は学年ロードマップ（B4=経験する・M=設計し完走する・D=定義し証明し言語化する）の上に置く。
