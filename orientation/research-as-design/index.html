LAB OS — CHAPTER 02

研究は設計で進める

Research as Design — 研究が止まる理由は能力ではない。順番と構造の欠如である

止まる原因はデータ不足ではない。方向性の欠如である

研究が停滞する瞬間を観察すると、共通点があります。データは増えている。解析も進んでいる。それでも、何を示したいのかが曖昧なまま——つまり問いが不在なのです。止まる原因は能力でもデータ不足でもなく、方向性の欠如です。

失敗パターン：データ先行

最も典型的な失敗は「データ先行」です。

  1. 🧪 とりあえず実験 — まず手を動かす
  2. 📊 とりあえず解析 — 出たデータを解析する
  3. ❓ あとで意味を考える — 方向なき探索、迷走のループへ

意味が後から立ち上がることを祈る——これは設計ではありません。探索の迷走です。地図を持たずに森に入るような研究は、今日で終わりにしましょう。

正しい順番：Framingが研究全体の設計図

正しい順番はこうです。

Idea → Framing ★ → Figures → Draft

Framing（構造化）が KEY STEP。下流のすべて——どんな図を作るか、何を書くか——は Framing から決まります。順番がすべてです。

Framingとは何か — 書くのは3つだけ

  1. 問いを1文で書く — Core Question in 1 sentence
  2. 先行研究との差分を明示する — "Compared to X, we Y."
  3. 貢献を最大3点に絞る — Key contributions (3 max)

注意：ここに結果を書いてはいけません。手法も書いてはいけません。Framingは「何を主張するか」の構造だけを固定する工程です。

Framingが弱いと、全て崩れる

✕ 曖昧な問い"We studied X and found something interesting..." — 何が新しいのか、条件は何かが不明。

○ 構造化された問い"Compared to prior method X, our approach Y achieves Z under condition C." — 差分・成果・条件が1文で明確。

問いは「Compared to X, we Y」の型で言い切る。言い切れないなら、Framingがまだ弱いということです。

良い研究の問いは、必ず「型」に収まる

問いの型問いの形
推定型 / EstimationHow much / many?
機構型 / MechanismHow / why does it work?
妥当性型 / ValidityDoes method X work?
不確実型 / UncertaintyWhat are the limits?
スケール型 / ScaleDoes it scale?
動態型 / DynamicsHow does it change?
設計型 / DesignHow should we build X?

自分の問いがどの型か選べないなら、その問いはまだ曖昧です。

問いが決まると、必要なデータが決まる

設計としての研究では、順序が逆転します。

  1. Question — 問いを確定する。構造がデータを呼び込む
  2. Required Data — 必要なデータが問いから逆算される
  3. Method / Experiment — 手法・実験。データは最後

✕Data → 意味が生まれることを祈る　○Question → Required Data

1図＝1主張 — 図は説明ではない。主張そのものである

BAD1つの図に主張Aと主張Bを詰め込む → 何を読み取ればいいのか分からない。

GOOD1つの図に主張は1つ。主張Bは次の図へ。

読者は論文を図で読みます。あなたも論文を読むとき、まず図から読んでください（Read papers via figures first）。

構造が生産性を決める — 能力差ではない。構造差である

High Structure：Framing → Figures → Draft → Submit。設計の質が成果を決める。
Low Structure：Data → ??? → Repeat。同じ時間でも成果が出ない。

同じ時間投資でも、構造が違えば成果が変わる。だから研究室として共通のOSが必要なのです。

Paper Card：構造を固定する装置

不確かなアイデア（Doubt）を確信ある設計（Belief）へ変えるために、鈴木研では Paper Card という1枚のカードを使います。書く項目は5つ：

  1. Core Question — 問いを1文で
  2. Compared to X, we Y — 先行研究との差分の明示
  3. Key Contributions — 貢献は3点以内
  4. Main Figures (sketch) — 図の配置スケッチ
  5. Contribution Statement — 貢献の要約

Structure before data — No Card → No Paper. カードなしに論文は始まりません。

この章のまとめ：止まる原因は問いの不在 ／ 順番は Idea → Framing → Figures → Draft ／ 問いは「Compared to X, we Y」で言い切る ／ 問いがデータを決める（逆ではない）／ 1図1主張 ／ Paper Card で構造を固定する。
